# Introduction

Welcome to **Gurukul**, a modern, full-stack SaaS boilerplate designed for high-performance and scalability. This documentation provides a comprehensive guide to understanding, setting up, and developing within the Gurukul ecosystem.

## The Gurukul Ecosystem

Gurukul is organized as a monorepo using Turborepo, bringing together various specialized applications and packages:

- **Web Application**: A high-performance dashboard built with Next.js (App Router) and Tailwind CSS v4.
- **API (Backend)**: A robust, scalable backend powered by NestJS and PostgreSQL.
- **Documentation**: This technical documentation site, integrated directly into the monorepo.

## Core Philosophies

1. **Modular Architecture**: Feature-based organization in the frontend and service-oriented patterns in the backend.
2. **Type Safety**: End-to-end TypeScript integration ensuring reliability and developer productivity.
3. **Developer Experience**: Optimized workflows with pre-configured linting, formatting, and automated checks via Husky.
4. **Cloud Native**: Fully containerized setup for easy local development and production deployment.

## Getting Started

To begin developing with Gurukul, proceed to the [Installation](/docs/getting-started/installation) guide to set up your local environment.
